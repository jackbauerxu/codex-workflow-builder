# 开箱与首用项目脚本

适用于用户实际拥有的包装、配件或第一使用场景。重点是“拿到什么、怎么用、谁需要”，而不是用空镜头制造稀缺感。

输入：`asset_manifest`、`package_contents`、`first_use_task`、`verified_spec_sources`、`missing_items`。

| 节拍 | 画面要求 |
| --- | --- |
| 期待与范围 | 包装正面或物件本体；说清本条只开哪些内容。 |
| 逐件揭示 | 每件配件出现一次，名称与用途同步。 |
| 首次设置 | 展示一个真实连接、安装或使用动作。 |
| 关键细节 | 特写一个会影响购买/使用决定的细节。 |
| 适合谁 | 给出适用条件与不适合的情况，再给一个 CTA。 |

输出字段：`included_items[]`、`first_use_steps[]`、`detail_to_check`、`missing_asset[]`、`fit_statement`。不能把未在素材里出现的配件、赠品、体验或规格说成已确认事实。
