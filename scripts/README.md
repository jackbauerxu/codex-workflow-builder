# 项目脚本库

基础骨架负责一致性：`opening.md` → 一个项目模板 → `storytelling.md` → `outro.md` → `captions.md`。项目模板负责“这条片子为什么值得看、如何证明、节奏怎样推进”。

这不是封闭分类。先按受众承诺、可用证据和叙事形态选择最接近的模板；遇到新题材时，保留基础骨架，复用最接近模板的证明方式，并在 Production Configuration 里写出 `custom beat map`。不要因为没有完全同名的模板就退回成泛泛的口播。

| 模板 | 适用项目 | 核心证明 |
| --- | --- | --- |
| `ai-demo.md` | AI/SaaS 功能演示 | 可复现的录屏任务 |
| `product-launch.md` | 产品发布、功能介绍 | 物件或界面中的真实使用瞬间 |
| `unboxing.md` | 开箱、首用、套装讲解 | 用户拥有的包装与配件 |
| `news-explainer.md` | 已确认的新闻、公告、政策解读 | 日期与一手来源 |
| `tiktok-shorts.md` | 观点、技巧、挑战、复盘 | 一个可验证的洞见或动作 |
| `b-roll-sequence.md` | 补足叙事、解释抽象句 | 与旁白一一对应的授权素材 |
| `how-to-tutorial.md` | 操作教程、方法论 | 可执行步骤与失败边界 |
| `case-study.md` | 项目复盘、客户案例、过程拆解 | 前后状态与可核对证据 |
| `comparison-decision.md` | 方案对比、选型、评测 | 同一标准下的事实对照 |
| `trend-commentary.md` | 行业变化、趋势判断、观点评论 | 已发生的信号与明确的不确定项 |

“爆款”不等于夸张承诺。每个模板都必须在前段交代冲突或收益、中段给出证明或动作、结尾只保留一个低摩擦下一步；无法证实的结果、素材或案例一律标记为 `unsupported` 或 `missing_asset`。
