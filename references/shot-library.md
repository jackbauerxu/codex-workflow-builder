# Shot Library

| ID | 用途 | 构图与动作 | 建议时长 |
| --- | --- | --- | --- |
| `hook-problem` | 开场冲突 | 一个具体错误、摩擦或前后状态，主体占 60% | 1.0-2.0s |
| `headline-card` | 新闻钩子 | 标题 + 日期/来源，数据不超过一个 | 1.5-2.5s |
| `screen-proof` | AI/产品证明 | 真实录屏关键区域 + 光标动作 | 2.0-4.0s |
| `feature-detail` | 产品细节 | 单一功能近景，旁注不超过两条 | 1.5-3.0s |
| `before-after` | 可验证对照 | 同一对象的旧/新状态，不伪造成果 | 2.0-3.0s |
| `process-path` | 方法解释 | 2-3 步路径，强调色沿路径推进 | 2.0-4.0s |
| `data-card` | 数据解释 | 一个图表或数字，显式标来源/单位 | 2.0-4.0s |
| `source-card` | 新闻来源 | 来源名称、日期、原话摘要，不伪造截图 | 1.5-2.5s |
| `context-broll` | 上下文 | 能证明场景的真实物件、手部或环境 | 1.0-2.5s |
| `human-reaction` | 使用感 | 非特定人物对真实结果的反应，不做夸张表演 | 1.0-2.0s |
| `result-state` | 收束 | 明确的产品/流程最终状态 | 1.5-3.0s |
| `cta-card` | 单一行动 | 一句下一步 + 干净留白 | 1.5-2.5s |

分镜优先调用这些 ID；需要新镜头时，补充同样的用途、画面、时长和禁用误导项，不能只写“来点 B-roll”。
